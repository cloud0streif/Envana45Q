# CCS Project Tracking Platform - Architecture Design

## Executive Summary

A web-based platform for tracking and monitoring Carbon Capture and Storage (CCS) operations with real-time IoT integration, document management, and hierarchical data visualization.

---

## Technology Stack Recommendation

### Frontend
- **Framework**: React.js or Vue.js
- **UI Library**: Material-UI or Tailwind CSS
- **State Management**: Redux or Vuex
- **Routing**: React Router or Vue Router
- **Data Visualization**: Chart.js or D3.js
- **Modal/Overlay**: React Modal or custom overlay components

### Backend
- **API Server**: Node.js with Express or Python FastAPI
- **Database**: PostgreSQL (structured data) + TimescaleDB (time-series sensor data)
- **File Storage**: AWS S3 or local file system with metadata DB
- **Authentication**: JWT tokens with role-based access control
- **WebSocket**: Socket.io for real-time sensor data

### IoT Integration
- **MQTT Broker**: Mosquitto (for sensor data streaming)
- **Data Pipeline**: Node-RED or custom Python scripts
- **Raspberry Pi Communication**: REST API + MQTT

### Deployment
- **Containerization**: Docker + Docker Compose
- **Web Server**: Nginx
- **Hosting**: Cloud (AWS/Azure/GCP) or on-premise

## Database Schema

### Tables

#### 1. **wells**
```sql
CREATE TABLE wells (
    well_id SERIAL PRIMARY KEY,
    well_name VARCHAR(100) NOT NULL,
    well_type VARCHAR(50) NOT NULL, -- 'injector', 'monitoring'
    location_lat DECIMAL(10, 8),
    location_lng DECIMAL(11, 8),
    depth_meters DECIMAL(10, 2),
    status VARCHAR(50), -- 'operational', 'maintenance', 'inactive'
    installation_date DATE,
    capacity_tpd DECIMAL(10, 2), -- tonnes per day
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. **sensor_data** (TimescaleDB hypertable)
```sql
CREATE TABLE sensor_data (
    time TIMESTAMPTZ NOT NULL,
    well_id INTEGER REFERENCES wells(well_id),
    sensor_type VARCHAR(50), -- 'temperature', 'pressure', 'humidity', 'flow'
    value DECIMAL(10, 4),
    unit VARCHAR(20),
    device_id VARCHAR(100), -- Raspberry Pi identifier
    PRIMARY KEY (time, well_id, sensor_type)
);

-- Convert to hypertable for time-series optimization
SELECT create_hypertable('sensor_data', 'time');
```

#### 3. **capture_facilities**
```sql
CREATE TABLE capture_facilities (
    facility_id SERIAL PRIMARY KEY,
    facility_name VARCHAR(100) NOT NULL,
    technology_type VARCHAR(100), -- 'amine', 'membrane', 'oxy-fuel'
    capacity_tpd DECIMAL(10, 2),
    location VARCHAR(200),
    status VARCHAR(50),
    efficiency_percent DECIMAL(5, 2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4. **transport_assets**
```sql
CREATE TABLE transport_assets (
    asset_id SERIAL PRIMARY KEY,
    asset_type VARCHAR(50), -- 'pipeline', 'truck', 'rail'
    asset_name VARCHAR(100),
    capacity_tpd DECIMAL(10, 2),
    status VARCHAR(50),
    specifications JSONB, -- Flexible storage for various asset properties
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 5. **documents**
```sql
CREATE TABLE documents (
    document_id SERIAL PRIMARY KEY,
    document_type VARCHAR(100), -- 'Class VI MRV', 'Permit', 'Report'
    title VARCHAR(255),
    description TEXT,
    file_path VARCHAR(500),
    file_size_kb INTEGER,
    uploaded_by INTEGER REFERENCES users(user_id),
    upload_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    version VARCHAR(20),
    category VARCHAR(50), -- 'capture', 'transport', 'sequestration'
    subcategory VARCHAR(50) -- 'injection', 'monitoring', etc.
);
```

#### 6. **users**
```sql
CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(100) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50), -- 'admin', 'operator', 'viewer'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP
);
```

#### 7. **alerts**
```sql
CREATE TABLE alerts (
    alert_id SERIAL PRIMARY KEY,
    alert_type VARCHAR(50), -- 'warning', 'critical', 'info'
    source_type VARCHAR(50), -- 'well', 'facility', 'pipeline'
    source_id INTEGER,
    message TEXT,
    severity VARCHAR(20), -- 'low', 'medium', 'high', 'critical'
    acknowledged BOOLEAN DEFAULT FALSE,
    acknowledged_by INTEGER REFERENCES users(user_id),
    acknowledged_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## API Endpoints

### Well Management
```
GET    /api/wells                    - List all wells
GET    /api/wells/:id                - Get specific well details
POST   /api/wells                    - Create new well
PUT    /api/wells/:id                - Update well information
DELETE /api/wells/:id                - Delete well

GET    /api/wells/:id/sensor-data    - Get sensor data for well
GET    /api/wells/:id/real-time      - WebSocket for real-time data
```

### Capture Operations
```
GET    /api/capture/facilities       - List capture facilities
GET    /api/capture/facilities/:id   - Get facility details
GET    /api/capture/performance      - Get performance metrics
POST   /api/capture/facilities       - Create new facility
```

```
GET    /api/transport/pipelines      - List pipelines
GET    /api/transport/vehicles       - List transport vehicles
GET    /api/transport/pipelines/:id  - Get pipeline details
GET    /api/transport/fleet-status   - Get fleet status
```

### Python Script for Raspberry Pi (Example)

```python
import time
import requests
import board
import adafruit_bme280

# Configuration
API_ENDPOINT = "https://your-platform.com/api/iot/sensor-data"
API_KEY = "your-api-key-here"
WELL_ID = 1  # Injector Well #1

# Initialize sensor
i2c = board.I2C()
bme280 = adafruit_bme280.Adafruit_BME280_I2C(i2c)

def send_sensor_data():
    """Read sensor and send to platform"""
    try:
        payload = {
            "well_id": WELL_ID,
            "timestamp": time.time(),
            "sensors": [
                {
                    "type": "temperature",
                    "value": bme280.temperature,
                    "unit": "celsius"
                },
                {
                    "type": "humidity",
                    "value": bme280.relative_humidity,
                    "unit": "percent"
                },
                {
                    "type": "pressure",
                    "value": bme280.pressure,
                    "unit": "hPa"
                }
            ]
        }
        
        headers = {
            "Content-Type": "application/json",
            "Authorization": f"Bearer {API_KEY}"
        }
        
        response = requests.post(API_ENDPOINT, json=payload, headers=headers)
        
        if response.status_code == 200:
            print(f"Data sent successfully: {payload}")
        else:
            print(f"Error: {response.status_code}")
            
    except Exception as e:
        print(f"Error sending data: {e}")

# Main loop
while True:
    send_sensor_data()
    time.sleep(10)  # Send data every 10 seconds
```
